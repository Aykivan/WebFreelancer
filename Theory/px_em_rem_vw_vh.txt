Абсолютные единицы - не зависят от значений других свойств. Фиксированные, окончательные.
   px - может быть дробным 
   font-size - наследуется потомками
     Преиммущества - четкость понятность ростота
     Недостатки - обьемная работа с адаптацией
   

Относительные - зависят от того или иного значения других свойств.
   em - размер формируется относительно текущего или унаследованного размера шрифта
   С каждым уровнем вложенности размер шрифта будет расти
      Преиммущства - упрощает адаптацию используется в медиа запросах, не ограничивает настройки размера шрифта браузером
      Недостатки - сложность в работе так как требует вычислений
   Используется для задания отступа вниз(делим размер по макету на размер шрифта) - авотоматический адаптив при изменении шрифта
      для заданаyия letter-spacing
      для медиазапросов
   Рекомендация использовать для медиа запросов и для тех свойств которые мы хотим привезять к текущему размеру шрифта

   rem - относительная единица, размер которой формируется от размера шрифта в теге html или браузера по умолчинию
      в отличие от em не важен шрифт родителя
   Используется для задания шрифтов в блоке body для самого часто используемого размера шрифта в макете, а так же для размеров
      контейнера
   Рекомендуется использовать для задания размеров шрифтов без внесения фиксированного размера шрифта в теге html 
      это даст возможность пользователю изменять шрифт с помощью настроек браузера
   Для вычисления единицы необходимо поделить нужную нам ширину на размер шрифта в макете
   
   % - оносительная единица. Разные css свойства вычисляют проценты от разных значений.
      width и height - относительно родителя, но только если эти значения у него заданы, или grid or flex
      pos: relativ - так же как и верхние два
      pos: abs - относительно родителя
      margin and padding - относитлеьно ширины родителя
      transform: translate - относительно обьекта к которому применяются
      font-size and line-hight - относительно унаследованного font-size
   Используется для декор элементов с pos abs для сохранения пропорций при адаптации
      ширину элеметна мы делим на ширину контейнера умножаем на 100 - width
      height - высоту картинки по макету делим на ширину контейнера умножаем на 100 и задаем в padding-top: 20%
      top: 50% опускаем обьект относительно обьекта в котором мы его позиционировали для выравнивания его по восоте 
      transform: rtanslate(50%, -50%) сдвегием вправо на 50% и выравниваем ее по оси y на -50%
      line-height в процентах(делим размер по макету на размер шрифта и умножаем на 100) в частях без умножения на 100
   Рекомендации: для построения отзывчивых резиновых обьектов, для позиционирования и скрола

   VH, VH, VMIN, VMAX - относительно окна браузера(вьюпорта)
   VH - относительно высоты, VW - относительно ширины
   VMIN/VMAX - относительно наименьшего/наибольшего значения среди VH VW
      Преимущества - адаптивная верстка
      Недостатки - высокий порог входа
   Рекомендации: для полноэкранных блоков и более сложных вычеслиний
